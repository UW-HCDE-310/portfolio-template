Japanese Dictionary and Flashcards

June 14th, 2019 Naomi Johnson

Project

Learning a new language involves looking up new words in a dictionary--- but sometimes students find themselves looking up the same word again and again if they do not learn it. This is particularly difficult for students learning Japanese. In addition to understanding the english meaning of a given word, they will need to learn the pronunciation of the word (written in either hiragana or katakana) as well as the written kanji (symbols that represent the meaning of the word, not the pronunciation).

Acomplishments

My team and I created a website that is not only a Japanese dictionary (complete with xyz thousand entries from the EDICT files from the Electronic Dictionary Research and Development Group at Monash University), but also lets users login to create flashcard decks, practice the flashcards, and record whether they knew the answer or not. We wrote a project proposal that summarizes the requirements, technologies, compares similar apps, and shows the user interface. I created a functioning prototype with AdobeXd to showcase the user experience. Unfortunately, our AWS machine has expired and needs to be set up again, but our our code is available.

For this project, I wrote Python scripts that converted the JSON files into MySQL statements ("meta programing") to fill our databases. I learned not everyone uses UTF-8, and I spent a long time working to get all the different coding languages to use Japanese successfully. I designed the flashcard functionality, my collaborator added the ability to sign up/login, and we collaborated on the dictionary functionality.

I created user personas, designed user scenarios, created wireframes, and made a functional prototype with AdobeXd. As features became functional on our live site, I designed qualitative user studies, ran them, and analyzed data. Even as we finished the final product, we were gathering data and creating action items that could improve the quality and usability of our site.

Results

We created a fully functional relational database with nine tables that had a combined total of 1.25 million entries, eleven thousand kanji, and tens of thousands of words, some of which had over ten different definitions. I earned an "A" on this project and in this course, as did my collaborator.

My collaborator and I created backend features to track whether users got flashcards right and how long they spent thinking about it, but we have not yet created an algorithm to determine how often users should be quizzed on their vocabulary words, how much they should be rewarded for immediately getting an answer correct, or how many times they need to get it right before we can take the word off the rotation for a while. We hope to create a front end that will display some of this data, as well as statistics about how many words they are studying per day.
