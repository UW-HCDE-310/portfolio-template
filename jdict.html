Jdict

Japanese Dictionary and Flashcards

Summary

This website is a fully-functioning Japanese-English dictionary that allows users to add any word they look up into flashcard decks. I worked with a team of three software engineers to start with low-fidelity sketches and do iterative usability tests as we developed the website.

Timeline: May-August 2019

Project

My team and I created a website that is a Japanese dictionary , that allows users to login to create flashcard decks, practice the flashcards, and record whether they knew the answer or not. We wrote a project proposal that summarizes the requirements, technologies, compares similar apps, and shows the user interface. I created a functioning prototype with AdobeXd to showcase the user experience. Unfortunately, our AWS machine has expired and needs to be set up again, but our code is available on GitHub.

For this project, I wrote Python scripts that converted the JSON files into MySQL statements (meta programing) to fill our databases. I learned not everyone uses UTF-8, and I spent a long time working to get all the different coding languages to use Japanese successfully. I designed the flashcard functionality, my collaborator added the ability to sign up/login, and we collaborated on the dictionary functionality.

User Studies

I created user personas, designed user scenarios, created wireframes, and made a functional prototype with AdobeXd. As features became functional on our live site, I designed qualitative user studies, ran them, and analyzed data. Even as we finished the final product, we were gathering data and creating action items that could improve the quality and usability of our site.

Acomplishments

In addition to writing the code and running the user studies, there are several other features in development. My collaborator and I created backend features to track whether users got flashcards right and how long they spent thinking about it, but we have not yet created an algorithm to determine how often users should be quizzed on their vocabulary words, how much they should be rewarded for immediately getting an answer correct, or how many times they need to get it right before we can take the word off the rotation for a while. We hope to create a front end that will display some of this data, as well as statistics about how many words they are studying per day.
